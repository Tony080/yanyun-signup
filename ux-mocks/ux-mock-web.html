问题：Web 版首次用户无引导

Web 版和小程序共享后端数据，但首次访问 Web 的用户不知道自己的身份是什么、恢复码是干什么的。直接显示 recovery code input 没有任何说明。

方案：Onboarding 三步引导

首次用户看到简单三步说明 + 设置身份表单 + "已有小程序账号？"恢复入口。3秒内理解产品是做什么的。

问题：桌面端浪费空间

max-width: 500px 在 1920px 屏幕上只用了 26% 的宽度。没有利用桌面端的优势。

方案：可选双栏布局

桌面端（>768px）使用双栏：左边是时段列表（主操作区），右边是热度图 + 自动报名 + 推荐卡片（信息侧栏）。把原本纵向堆叠的内容平铺，减少滚动。手机端保持单列不变。

作为"快速版小程序"的定位

既然 Web 版是因为小程序审核慢的替代方案，核心体验应该和小程序完全一致。所有 Mock 1-4 的改进都应同步到 Web 版。桌面端双栏是额外加分项，不是必须的。
