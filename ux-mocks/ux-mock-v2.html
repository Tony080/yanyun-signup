10人和1人的柱子几乎一样高。原因：min-height: 6% 太大，且用的是百分比而非对数/绝对高度。1人时 max(6%, 1/10*100%) = 10%，和 100% 相比差距不够。

问题 2：职业每次默认输出，没有记忆

用户每次报名都默认"输出"，霖霖玩家每次都要手动切换。应该在角色名旁固定选择并自动记忆（存 localStorage / 云端）。

问题 3：底部文字突兀

"每周自动刷新 | 每人每周限报一个时段 | 10人满开新车 | 时间已自动转换为你的本地时区" — 放在页面最底部很突兀，像 debug 信息。需要重新考虑放置位置和呈现方式。

柱状图：每人 = 固定 4px 高度

不再用百分比。1人=4px，10人=40px，30人=120px。0人=2px 灰条。对比一目了然，不会出现"1人和10人差不多高"的问题。

职业：贴在昵称右侧

紧凑一行：[角色名] [昵称输入框] [🔵输出]。点击可切换，选择后自动记忆到 localStorage。报名时直接用已选职业，不再每次默认输出。

规则文字：融入 header 底部

一行小字贴在 header 底部分割线下，替代占整行的 footer。简洁、不突兀、不占额外空间。

柱状图：对数缩放

用 log(count+1) 映射高度。0→2px, 1→7px, 4→22px, 10→50px, 30→70px。低人数不会塌成一条线，高人数也不会撑爆容器。

职业：Header 内双按钮行

在昵称下方放两个全宽按钮。像 tab 切换一样高亮当前选择。下方提示"选好后自动记忆"。优势：触控面积大，不易误触。

规则：可关闭的信息条

蓝色信息条带 × 关闭按钮。新用户首次看到后可关闭，下次不再显示（记忆到 localStorage）。不影响老用户的界面。

柱状图：5px/人 + 60px 封顶

每人贡献 5px 高度，最大不超过 60px。0人=2px。优势：线性直觉、简单好理解。1人=5px vs 10人=50px，对比度清晰。超过12人后封顶，数字已经标在上方了。

职业：双色圆点

两个紧凑圆点 🔵🟢 贴在昵称输入框右侧。选中的高亮+边框，未选的半透明。极省空间，适合已熟悉系统的用户。

规则：首次 tooltip

紫色提示条，只在首次使用时显示。用户关闭后通过 localStorage 记住。同时 "本地时区" 四个字融入 header 副标题。没有 footer 了。
