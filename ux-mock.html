Current UX Pain Points
Based on review of Web, WeChat Mini Program, and Discord interactions

1. Header Information Overload

Header area packs too many functions: nickname editing, proxy signup input, recovery code display. First-time users can't quickly understand "what should I do?"

Web Mini Program

2. Unclear Entry Points

"Quick Join" vs "Create Team" -- the difference is not obvious to new users. No onboarding explains the two modes. Users unsure which button to press.

Web Mini Program Discord

3. Poor Time Slot Readability

9 time slots displayed flat with high info density but no visual hierarchy. Can't quickly see which slot is popular or has availability.

Web Mini Program

4. Inconsistent Cross-Platform UX

Discord uses multi-step serial interaction (mode -> time -> role -> auto), non-reversible. Web/Mini use modal one-shot. No unified visual language.

Discord

5. Weak Car Visualization

Progress bar is only 3px -- barely visible. Members are tag-style with no sense of team composition (DPS/Healer ratio). No guidance on "which car needs what role?"

Web Mini Program

6. Excessive Confirmation Friction

Default name confirmation requires manually typing name back -- too much friction. Leave/move lacks quick actions. Proxy add flow is clunky.

Web Mini Program

1. Status Card at Top

User's own registration status always visible at the top. No need to scroll through 9 time slots to find "where am I." Move/Leave buttons inline.

2. Popularity Heatmap

Visual bar chart replaces flat time slot list. Color + height encode slot popularity. Users instantly see which times are hot (amber) vs available (green) vs empty (gray).

3. Smart Recommendation

System analyzes team composition gaps and recommends a slot + role. "This car needs 2 healers" -- reduces decision fatigue for casual players.

4. Compact Slot List

Collapsed by default: time + count + action. Expand for car details. Reduces visual clutter from showing all car cards upfront.

5. Contextual CTA

Full slots show "Create Car" instead of "Join." No more clicking "Join" only to be told "this slot is full."

1. Onboarding Card

First-time users see a guided intro explaining the 3 core concepts. Dismissible with dots indicator for multi-page walkthrough. Clears the "I don't know which button to press" problem.

2. Swipeable Time Cards

Replace the dropdown/flat list with visual cards. Each card shows time, people count, capacity bar, and percentage. Selected card highlighted with blue border. Feels more interactive.

3. Team Composition Hints

"This car needs 2 more healers" shown when selecting role. Helps players make better team decisions, not just pick their default.

4. Explicit CTA Button

"以输出加入 周日 3PM" -- the button text says exactly what will happen. No ambiguity. No "which mode am I in?" confusion.

5. Inline Proxy (Bring Friends)

Proxy signup moved from header to bottom of signup flow. Makes sense contextually: "I'm joining, and bringing friends with me." Reduces header clutter.

1. Bubble Timeline

Each time slot is a bubble. Size = people count, Color = popularity tier. Tap to select. Far more scannable than 9 identical-looking slot blocks.

2. Role Ratio Donut

Each car shows a donut chart of DPS (blue) vs Healer (green) ratio. Instantly see team composition balance without counting tags. Critical for players who care about optimal team makeup.

3. Thicker Progress Bars

8px bars with percentage text. Current 3px bars are nearly invisible. "80% -- needs 2 more" is much more actionable than just a thin line.

4. Bottom Navigation

Separate "Signup", "My Status", "Settings" into tabs. Currently everything is crammed into one scrollable page. Settings = nickname, recurring, proxy management, recovery code.

5. Sticky Join Bar

Fixed bottom CTA always accessible. Users can browse cars, scroll through members, and still one-tap join without scrolling back up.

Common Improvements (All Proposals)
These changes should be applied regardless of which design direction is chosen

1. Move Proxy Signup Out of Header

Proxy signup ("help others register") should be at the bottom of the signup flow, not in the header. It's a secondary action that clutters the primary view.

BEFORE
Header: Title + Week + Nickname + Proxy Input + Recovery Code
= 5 features in one area
AFTER
Header: Title + Week + Nickname only
Proxy: Bottom of signup flow
Recovery: Settings tab

2. Thicken Progress Bars (3px -> 8-10px)

Current car progress bars are barely visible. Increase to 8-10px with percentage text alongside.

BEFORE
(Can you even see that?)
AFTER
60% -- 还差4人

3. Simplify Default Name Confirmation

Replace "type your name to confirm" with a simple two-button dialog. The current flow has too much friction.

BEFORE
Modal: "Please type 水仙十字社小可爱abc below to confirm"
[text input]
[Cancel] [Confirm (disabled until match)]
AFTER
Modal: "Your name is 水仙十字社小可爱abc"
"Want to change it before signing up?"
[Change Name] [Continue]

4. Discord: Add Back Button

Current Discord flow is 4 irreversible steps. Add a "Back" button at each step so users can correct mistakes.

BEFORE
Mode -> Time -> Role -> Recurring
(Can't go back! Wrong time? Start over.)
AFTER
Mode -> Time -> Role -> Recurring
[← Back] available at each step
Or: combine into fewer steps

5. Smart "Create Car" CTA for Full Slots

When all cars in a slot are full, change the "Join" button to "Create New Car" automatically.

BEFORE
User clicks "Join" on full slot
-> System auto-creates new car
(User surprised: "I wanted to join existing car")
AFTER
Full slot shows:
[Create New Car] instead of "Join"
Clear expectation before clicking
